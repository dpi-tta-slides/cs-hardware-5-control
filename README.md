# CS Hardware Day 5 - Control

In this lesson, participants connect the previous four days of hardware concepts into a complete computing system. Building on transistors, timing circuits, oscillators, and integrated circuits, participants learn how a microcontroller places an entire computer onto a single chip. Using the BBC micro:bit, participants explore digital inputs and outputs, sensors, LEDs, buttons, and simple programming. Through a series of increasingly sophisticated projects, participants connect software instructions to physical hardware behavior and experience how modern embedded systems interact with the real world. The lesson emphasizes the relationship between hardware and software, input/output systems, sensors, timing, debugging, and physical computing.

[Slides](https://dpi-tta-slides.github.io/cs-hardware-5-control/)

## Learning Objectives

By the end of this lesson, participants will be able to:

- Explain what a microcontroller is
- Describe how microcontrollers relate to integrated circuits
- Identify major microcontroller subsystems:
  - CPU
  - Memory
  - Inputs
  - Outputs
  - Sensors
- Explain the difference between a microcontroller and a general-purpose computer
- Program a BBC micro:bit
- Use buttons as digital inputs
- Control LEDs as digital outputs
- Use loops and conditional logic
- Read data from onboard sensors
- Build interactive physical computing projects
- Connect previous hardware concepts to embedded systems
